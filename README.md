# せどりリサーチツール

キーワードを入力すると各仕入れサイトを自動検索し、Amazon相場と照らし合わせて利益計算するPythonツールです。

---

## 設計概要

```
main.py  ─── オーケストレーター
  ├── scraper.py  ─── Claude A（仕入れリサーチ担当）
  │     メルカリ / ヤフオク / Yahoo!フリマ
  │     オフモール / セカンドストリート / トレファク
  └── amazon.py   ─── Claude B（利益分析担当）
        Amazon価格取得 / FBA手数料計算 / 利益率判定
```

---

## インストール

```bash
cd ~/Desktop/sedori-tool
pip install -r requirements.txt
```

---

## 使い方

### 対話モード（推奨）

```bash
python main.py
```

起動後、プロンプトに従ってキーワードを入力します。
ASINコードや取得件数もその場で設定できます。

### コマンドライン引数モード

```bash
# キーワード検索
python main.py --keyword "Nintendo Switch"

# キーワード + ASIN指定（より正確な価格取得）
python main.py --keyword "Switch" --asin B07WGTMDKX

# 1サイト5件取得 + 結果をJSONに保存
python main.py -k "ゲームボーイ" -l 5 --save
```

#### オプション一覧

| オプション | 短縮 | 説明 |
|-----------|------|------|
| `--keyword` | `-k` | 検索キーワード |
| `--asin` | `-a` | Amazon ASINコード |
| `--limit` | `-l` | 1サイトあたりの取得件数（デフォルト: 3） |
| `--save` | `-s` | 結果をJSONファイルに保存 |

## 出力の見方

```
評価  仕入れサイト        商品名                           仕入値   Amazon  手数料     利益   利益率
─────────────────────────────────────────────────────────────────────────────────
 ◎   メルカリ            Nintendo Switch 本体セット…      ¥12,000  ¥32,000  ¥5,100   +¥14,900   46.6%
 ○   ヤフオク            ニンテンドースイッチ 中古…        ¥18,000  ¥32,000  ¥5,100   +¥8,900    27.8%
 △   オフモール          Switch ジョイコン付き…           ¥28,000  ¥32,000  ¥5,100   -¥1,100    -3.4%
 ✗   セカンドストリート  スイッチ本体のみ…               ¥31,000  ¥32,000  ¥5,100   -¥4,100   -12.8%
```

### 評価グレード

| マーク | 利益率 | 説明 |
|--------|--------|------|
| ◎ | 15%以上 | 優良案件。積極的に仕入れ検討 |
| ○ | 5%以上 | 良案件。仕入れ価格交渉余地あり |
| △ | 0%以上 | ギリギリ黒字。コンディション次第 |
| ✗ | マイナス | 赤字。この価格では仕入れ不可 |

## 手数料の計算方式

```
手数料 = Amazon販売価格 × 15%（カテゴリ手数料） + ¥300（FBA配送料概算）
利益   = Amazon販売価格 − 仕入れ価格 − 手数料
利益率 = 利益 ÷ Amazon販売価格 × 100
```

> カテゴリによって実際の手数料は異なります。
> ゲーム: 8%、電子機器: 8〜15%、書籍: 15%など。
> 詳細は [Amazon出品者向け手数料一覧](https://sellercentral.amazon.co.jp/gp/help/external/G200336920) を参照してください。

## スクレイピングについて

| サイト | 取得方式 | 備考 |
|--------|----------|------|
| メルカリ | 公開API | 最も安定 |
| ヤフオク | HTMLスクレイピング | HTML構造変更で要メンテ |
| Yahoo!フリマ | HTMLスクレイピング | JS描画のためモック併用 |
| オフモール | HTMLスクレイピング | セレクター要確認 |
| セカンドストリート | HTMLスクレイピング | セレクター要確認 |
| トレファク | HTMLスクレイピング | セレクター要確認 |
| Amazon | HTMLスクレイピング（モバイルUA） | ブロックされる場合あり |

**`[モック]` と表示されているデータは**、スクレイピングに失敗した場合の代替データです。
キーワードのハッシュ値から生成されるため、実際の相場とは異なります。参考値としてご利用ください。

## 注意事項

- 各サービスの**利用規約**を遵守してください
- 過度なリクエストは避け、適切な待機時間（1〜2.5秒）を設けています
- Amazonのスクレイピングは規約上グレーゾーンです。商業利用の場合はPA-API（Product Advertising API）の使用を推奨します
- スクレイピング先のHTML構造が変更された場合、セレクターの修正が必要になることがあります
